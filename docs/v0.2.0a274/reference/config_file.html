Configuration file#

Configurable settings that can be set within the configuration file are listed here.

Default config file#

configs:
  default:
    invocation:
      environment_setup:
      match: {}
    config:
      machine: HOSTNAME
      log_file_path: logs/hpcFlow_v0.2.0a274.log
      environment_sources: []
      task_schema_sources: []
      command_file_sources: []
      parameter_sources: []
      default_scheduler: direct
      default_shell: bash
      schedulers:
        direct:
          defaults: {}
      shells:
        bash:
          defaults: {}
      user_affiliations: []

Validation schema#

The configuration file must conform to the following validation schema.

type: dict (key: str) (value: dict)
Condition: ConditionAnd(ValueDataType.equal_to(value=<class 'dict'>), Value.keys_is_instance(<class 'str'>))

A mapping of string config "invocation keys" to configuration settings.

program_manifest_file#
type: str | path, optional
Condition: Value.is_instance(<class 'str'>, <class 'pathlib._local.Path'>)

If provided, this will be used as the path to the manifest for the built-in example programs. It should point to a JSON file whose top-level keys are example program file keys, like "hello_world/macos_arm64/hello_world". Each value should be an empty map/dict or a map like: {"in_zip": "hello_world.zip"}, where hello_world.zip is the name of a zip archive that contains the program. This can be a local path or an fsspec URL.

program_dir#
type: str | path, optional
Condition: Value.is_instance(<class 'str'>, <class 'pathlib._local.Path'>)

If provided, this will be used as the path to the directory containing built-in programs. This can be a local path or an fsspec URL.

parameter_sources#
type: list (of: str | path), required
Condition: ValueDataType.equal_to(value=<class 'list'>)
machine#
type: str, required
Condition: ValueDataType.equal_to(value=<class 'str'>)

A label that references the current machine. By default, this uses the return from the Python function socket.gethostname().

log_file_path#
type: str | path, optional
Condition: Value.is_instance(<class 'str'>, <class 'pathlib._local.Path'>)

File path to the app log file. The variables <<app_name>> and <<app_version>> are available to be used in this option, which resolve to the app name and version, respectively.

The file path may be nested in non-existent directories, in which case those directories will be generated. If specified as a relative path, the path will be resolved relative to the config directory (the directory that contains the config file).

log_file_level#
type: str, ('notset' | 'debug' | 'info' | 'warning' | 'error' | 'critical'), optional
Condition: ConditionAnd(ValueDataType.equal_to(value=<class 'str'>), Value.in_(value=['notset', 'debug', 'info', 'warning', 'error', 'critical']))

type: str | path, optional

If provided, this will be used as the path to the manifest for the built-in example data files. It should point to a JSON file whose top-level keys are example data file names. Each value should be an empty map/dict or a map like: {"in_zip": "text_file.zip"}, where text_file.zip is the name of a zip archive that contains the example data file. This can be a local path or an fsspec URL.
